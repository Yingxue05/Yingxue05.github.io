              • 单片机-定时器

              核心作用

              • 代替软件延时，无需占用CPU。
              • 产生精确时间间隔，实现定时控制。
              • 统计外部事件脉冲数。

              第一节 定时器概述

              定时/计数器组成：本质是16位加1计数器（由THx高8位合TLx低8位组成）。

              计数触发源：

              • 定时模式(C/T = 0)：计数器内部机器周期脉冲（在12MHz晶振频率下，机器周期为1μs），

                计数频率=1机器周期=晶振频率12

              • 计数模式(C/T = 1)：计数器外部引脚（T0：P3.4，T1：P3.5）的负跳变脉冲。

                最高计数频率=晶振频率24即需要满足：外部周期>2个机器周期

              第二节 定时/计数器的控制

              模式控制寄存器TMOD

              • 其高四位用于控制定时/计数器T1，低4位用于控制定时/计数器T0。

              • GATE（门控位）：

                • GATE = 1内部启动：TCON中对应TRx位也为1时，定时/计数器启动。
                • GATE = 0外部启动：TRx为1且外部中断引脚INTx也为1时，定时/计数器启动。

              • C/T¯（功能选择位）：

                • C/T¯=0定时器模式：计数内部机器周期。
                • C/T¯=1计数器模式：计数外部引脚负跳变脉冲。

              • M1,M0（模式选择位）：二者的0，1组合设置定时/计数器的四种工作模式。

              控制/状态寄存器TCON（仅列出与定时器相关位）

              • T1/0溢出标志位：溢出时硬件置1。响应中断后硬件自动清0，查询方式下需软件清零。

              第三节 定时/计数器的四种工作方式

              四种方式的对应关系

              模式0（13位定时/计数器）

              • 结构：由THx（8位）+TLx（低5位）组成13位计数器。
              • 原理：TLx的低5位溢出时向THx进位，THx溢出时，置位TCON中的TFx标志，向CPU发送中断请求。
              • 特点：兼容早期MCS-48架构，功能可被模式1替代，极少使用。

              模式1（16位定时/计数器）

              • 结构：由THx（8位）+TLx（8位）组成完整16位计数器。
              • 特点：定时范围广，精度适中，为最常用的模式。
              • 注意：溢出后需手动重装初值，查询方式使用JBC TFx检测溢出，中断方式在ISR中重装。

              模式2（8为自动重载模式）

              • 结构：TLx为8为计数器，THx存储初值，TLx溢出时自动将THx值重装到TLx中，无需软件重装初值。
              • 特点：定时精度高，无需手动重装，适合高频精确定时（串口通信波特率发生器）。

              模式3（T0拆分模式）

              • 结构：

                • 仅T0支持，此模式下将T0拆为两个独立的8位计数器。TL0占用T0的TR0、TF0；TH0只支持定时模式（不可计数）占用T1的TR1、TF1。
                • T1在模式3下停止工作，此时可用于串口波特率发生器。

              • 适用场景：需要两个独立定时器时（如实现两个不同频率的方波）。

              第四节 定时器初值的计算

              最大计数值：即为x位计数器的最大值2x。例如16位计数器最大计数值65536(216)。

              定时模式

              定时时间t=(最大计数值N−初值X)×机器周期T推导得到：初值X=N−tT

              计数模式

              初值X=最大计数值−目标计数值M（需计数的外部脉冲数）

              第五节 汇编实现示例

              查询方式（简单场景，占用 CPU

              • 核心流程：配置 TMOD→装载初值→启动定时器→查询 TFx→溢出处理→重装初值（模式 0/1）→循环。
              • 汇编示例（T0 模式 1，定时 50ms，P1.0 输出方波）：
              ORG 0000H
              AJMP MAIN
              ORG 0100H
              MAIN:
                  MOV TMOD, #01H   ; T0模式1，定时模式
                  MOV TH0, #3CH    ; 装载初值（50ms）
                  MOV TL0, #0B0H
                  SETB TR0         ; 启动T0
              LOOP:
                  JBC TF0, NEXT    ; 查询溢出，溢出则清TF0并跳转
                  SJMP LOOP        ; 未溢出则等待
              NEXT:
                  MOV TH0, #3CH    ; 重装初值
                  MOV TL0, #0B0H
                  CPL P1.0         ; P1.0电平翻转（方波输出）
                  SJMP LOOP        ; 循环
              END

              中断方式（高效，支持多任务）

              • 核心流程：配置 TMOD→装载初值→开总中断 + 定时器中断→启动定时器→编写 ISR（溢出处理 + 重装初值）。
              • 汇编示例（T0 模式 1，定时 1ms，P1.0 闪烁）：
              ORG 0000H
              AJMP MAIN
              ORG 000BH           ; T0中断入口地址
              AJMP T0_ISR
              ORG 0100H
              MAIN:
                  MOV TMOD, #01H   ; T0模式1，定时模式
                  MOV TH0, #0FCH   ; 初值=65536-1000=64536=0xFC18H
                  MOV TL0, #18H
                  SETB EA          ; 开总中断
                  SETB ET0         ; 开T0中断
                  SETB TR0         ; 启动T0
                  SJMP $           ; 主循环等待中断
              T0_ISR:
                  PUSH ACC         ; 保护现场
                  MOV TH0, #0FCH   ; 重装初值
                  MOV TL0, #18H
                  CPL P1.0         ; 翻转P1.0
                  POP ACC          ; 恢复现场
                  RETI             ; 中断返回（不可用RET）
              END
